Reference Architecture: Smart Return Assistant

Here is a reference design for the Day 12 mini-project. There is no single correct answer, but this approach demonstrates best practices using Week 2 concepts.

1. Orchestration: Single Agent vs. Steering Agent

Recommendation: For this specific scenario, a **Single Agent with focused tools** is likely sufficient to start, as the task is linear (Check policy -> Calculate refund -> Generate label). However, if EcoShop plans to add more features (like replacement ordering or customer support chat), transitioning to a **Steering Agent** would be better to separate return logic from general chat.

2. Tool Design

  • Knowledge Base Tool: A tool to query the return policy (e.g., query_policy(item_type)).
  • Python Code Tool (Deterministic Logic): A tool to calculate the refund. It takes the original price, condition of the item, and applies the policy rules (e.g., deducting a 10% restocking fee if applicable). 
    def calculate_refund(price, condition, applies_restocking_fee):
    if condition != "new" or applies_restocking_fee:
        return price * 0.9
    return price
  • Connector Tool (External API): An OpenAPI-based tool to generate the shipping label (e.g., {@TOOL: create_shipping_label} taking name, address, and package details).

3. Latency Optimization

To handle the combined request "Can I return this and how much will I get back?" efficiently:

  • Wrapper Pattern: Create a wrapper tool called check_return_eligibility_and_value(item_id, condition) that internally queries the policy and calculates the refund in one go, reducing turns.
  • Instruction Set: Alternatively, instruct the model to fetch the policy and calculate the refund in parallel or sequence before responding to the user, ensuring the user gets a complete answer in the first response turn.

Knowledge Check Answers

  1. B - The Wrapper Tool Pattern is specifically designed to reduce latency by combining multiple backend operations into a single execution turn.
  2. B - Python Code Tools are used to ensure deterministic, accurate results for operations where LLMs are prone to error, like math and complex logic.
  3. B - The main purpose of the Steering Agent Pattern is to manage complexity by delegating tasks to specialized components, preventing prompt overload in a single agent.
