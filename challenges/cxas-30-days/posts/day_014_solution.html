Reference Solution: Test Plan for `{@TOOL: apply_loyalty_discount}`

Here is a reference example of how you might structure your isolation test plan for the loyalty discount tool. This can be implemented using standard unit testing frameworks like `pytest` or `Jest`.

Reference Test Plan (YAML Structure)

# test_apply_loyalty_discount.yaml

- test_case_id: TC_LOYALTY_001_STANDARD_GOLD
  description: "Verify 10% discount for Gold tier customer."
  input:
    customer_id: "CUST_GOLD_123"
    cart_total: 100.00
  mock_data:
    loyalty_service.get_tier: "Gold"
  expected_output:
    discounted_total: 90.00
    discount_applied: 10.00

- test_case_id: TC_LOYALTY_002_INVALID_CUSTOMER
  description: "Verify behavior when customer ID is not found (should apply no discount)."
  input:
    customer_id: "CUST_NONEXISTENT"
    cart_total: 100.00
  mock_data:
    loyalty_service.get_tier: null # Or raises UserNotFoundError
  expected_output:
    discounted_total: 100.00
    discount_applied: 0.00

- test_case_id: TC_LOYALTY_003_ZERO_TOTAL
  description: "Verify behavior when cart total is zero."
  input:
    customer_id: "CUST_GOLD_123"
    cart_total: 0.00
  mock_data:
    loyalty_service.get_tier: "Gold"
  expected_output:
    discounted_total: 0.00
    discount_applied: 0.00

Prompting for Text Output Validation (If Applicable)

If the `{@TOOL: apply_loyalty_discount}` tool also generates a text explanation for the user (e.g., using an internal LLM call to make it sound friendly), you might need an LLM to evaluate that text. Here is how you should structure the evaluation prompt using best practices:

You are a QA evaluator for the EcoShop Agent system. Your task is to validate the explanation generated by the loyalty discount tool.


Analyze the generated explanation to ensure it accurately reflects the applied discount and maintains a positive, brand-aligned tone.



1. Verify that the explanation explicitly mentions the correct loyalty tier (e.g., Gold, Silver).
2. Check that the mentioned discount amount matches the calculated discount (e.g., $10.00).
3. Ensure the message does not reveal internal system details or raw JSON data.
4. Rate the message as PASS or FAIL with a short rationale.


Input Data:
- Customer Tier: Gold
- Discount Applied: $10.00
- Cart Total: $100.00

Generated Explanation:
"Great news! As a Gold member, we've applied a 10% discount to your order, saving you $10.00. Your new total is $90.00."

Knowledge Check Answers

  2. C - Callbacks are tested by manually invoking them with simulated event data and checking if the expected side effects (like database updates or alerts) occurred.
  3. B - Mocking is used to simulate external dependencies (like APIs or databases) to make tests fast, reliable, and deterministic.
