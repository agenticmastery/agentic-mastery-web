Solution: Python Script for Programmatic Build

Here is the complete Python script to create the "Sustainability Advisor" agent and the {@TOOL: get_carbon_footprint} tool. Note how we use a structured approach for the agent's instructions to ensure best practices.

import os
from cxas_scrapi import Apps, Agents, Tools

def build_sustainability_advisor():
    """Programmatically creates the Sustainability Advisor agent and a dummy tool."""
    project_id = os.environ.get("GCP_PROJECT_ID")
    app_id = os.environ.get("CXAS_APP_ID") # Assuming we have the App ID stored
    
    if not project_id or not app_id:
        print("Error: GCP_PROJECT_ID or CXAS_APP_ID environment variables not set.")
        return

    # Construct the full app resource name
    # Typically looks like: projects/{project}/locations/{location}/apps/{app}
    # For simplicity, we assume the full resource path is provided or easily constructed
    app_name = f"projects/{project_id}/locations/global/apps/{app_id}"

    try:
        agents_client = Agents()
        tools_client = Tools()

        print(f"Provisioning resources in App: {app_name}")

        # 1. Create the Tool first (so we can reference it in instructions if needed)
        # Best Practice: Use the {@TOOL: tool_name} syntax in documentation/references
        print("Creating tool: get_carbon_footprint...")
        tool = tools_client.create_tool(
            parent=app_name,
            display_name="get_carbon_footprint",
            type="code",
            code="""
def get_carbon_footprint(product_id: str) -> dict:
    # Dummy implementation
    return {"product_id": product_id, "carbon_score": "Low", "equivalent_miles_driven": 1.2}
"""
        )
        print(f"Tool created successfully: {tool.name}")

        # 2. Create the Agent with structured instructions
        print("Creating agent: Sustainability Advisor...")
        
        # Prompt examples in solution use the best practices structure
        instructions = """

You are the EcoShop Sustainability Advisor. Your goal is to help users understand the environmental impact of their purchases and guide them toward eco-friendly choices.



Handle inquiries about product carbon footprints.

When a user asks about the carbon footprint of a product, use the {@TOOL: get_carbon_footprint} tool.


Summarize the result for the user, explaining what 'Low', 'Medium', or 'High' impact means.




Suggest eco-friendly alternatives.

If a user is looking at a high-impact product, suggest a lower-impact alternative from the same category if known.


"""

        agent = agents_client.create_agent(
            parent=app_name,
            display_name="Sustainability Advisor",
            instructions=instructions.strip()
        )
        print(f"Agent created successfully: {agent.name}")

        print("\nProvisioning Complete!")
        print(f"Agent Resource Name: {agent.name}")
        print(f"Tool Resource Name:  {tool.name}")

    except Exception as e:
        print(f"An error occurred: {e}")

if __name__ == "__main__":
    # For testing, you would set these in your terminal:
    # export GCP_PROJECT_ID="your-project-id"
    # export CXAS_APP_ID="your-app-id"
    build_sustainability_advisor()

Key Steps Explained

  • Dependency Management: We create the tool first. While CXAS allows you to reference tools in instructions by name before they are created in some cases, creating them first is generally safer if the SDK validates references immediately or if you need the resource name.
  • Structured Instructions: We used <goal>, <subtask>, and <step> tags in the agent's instructions. This XML-like structure helps the LLM parse its role and execution steps more reliably than flat text.
  • Tool Syntax: We referenced the tool using the specific syntax {@TOOL: get_carbon_footprint} in the instructions, aligning with CXAS expectations.
