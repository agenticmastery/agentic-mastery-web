The Solution: Callback Strategy for EcoShop

Scenario 1: Enforcing Disclaimer on the First Turn

To ensure the disclaimer is shown on the very first turn, the most robust approach is to use an after_model_callback. This allows the model to generate its natural response first, and then we ensure the disclaimer is prepended if missing.

Logic for after_model_callback (Pseudocode)

def after_model_callback(session_state, model_response):
    # Check if this is the first turn of the conversation
    if session_state.turn_count == 1:
        disclaimer = "Notice: All returns must be initiated within 30 days of purchase."
        
        # Check if the model already included the disclaimer (case-insensitive)
        if disclaimer.lower() not in model_response.text.lower():
            # Prepend the disclaimer to the model's response
            model_response.text = disclaimer + "\n\n" + model_response.text
            
    return model_response

Scenario 2: Enforcing Disclaimer Before Answering Return Questions

If the requirement is to show the disclaimer specifically before answering questions about returns, we can use a specialized **Returns Sub-Agent** and attach the callback to it, or use a before_model_callback to inspect the input.

Here is how you would structure the prompt for the Returns Sub-Agent using the best practices we've learned, assuming the callback handles the actual insertion of the disclaimer but the prompt guides the model to be aware of it or use tools correctly.

<role>
You are a specialized customer service agent for EcoShop focused exclusively on handling returns, refunds, and exchanges.
</role>

<persona>
You are helpful, understanding, and efficient.
</persona>

<primary_goal>
Guide the customer through the return or refund process accurately and efficiently, ensuring they are aware of the policy.
</primary_goal>

<taskflow>
  <subtask name="Handle Return Query">
    <step>
      <trigger>Customer asks about return policy or starts a return.</trigger>
      <action>Acknowledge the request. Call the {@TOOL: check_return_eligibility} tool to verify status if order details are provided, then provide instructions. Note: A system callback will enforce the mandatory disclaimer.</action>
    </step>
  </subtask>
</taskflow>

<constraints>
- Do not handle questions about general product availability.
- Adhere to the 30-day return policy strictly.
</constraints>

Pro-Tip: Leverage Session Variables

Use session variables to track whether the disclaimer has already been shown. This prevents annoying the customer by repeating the disclaimer in every turn if they continue to ask about returns. Set a flag like disclaimer_shown = true in the callback after the first time it triggers.
