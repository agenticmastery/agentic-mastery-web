Detailed Lesson: CRUD Operations with SCRAPI

SCRAPI provides dedicated clients for different resource types in CX Agent Studio. The most commonly used are Apps, Agents, and Tools. Let's look at how to perform CRUD operations with them.

1. Create Operations

To create resources, you use the create_* methods on the respective clients. You typically need to provide a parent resource name or ID.

from cxas_scrapi import Apps, Agents, Tools

apps_client = Apps()
agents_client = Agents()
tools_client = Tools()

PROJECT_ID = "your-project-id"

# 1. Create an App
# Note: Apps are top-level resources in a project
new_app = apps_client.create_app(
    project_id=PROJECT_ID,
    display_name="EcoShop Support"
)
app_name = new_app.name # This is the full resource path

# 2. Create an Agent within the App
new_agent = agents_client.create_agent(
    parent=app_name,
    display_name="Inventory Bot",
    instructions="You help users find products in stock."
)

# 3. Create a Tool within the App
# Here we use a correct tool syntax reference in the concept: {@TOOL: check_inventory}
new_tool = tools_client.create_tool(
    parent=app_name,
    display_name="check_inventory",
    type="code",
    code="def check_inventory(product_name): return 5"
)

2. Read Operations

To retrieve details about a resource, use the get_* methods. To list resources, use list_*.

# Get specific agent details
agent = agents_client.get_agent(name=new_agent.name)
print(f"Agent Instructions: {agent.instructions}")

# List all tools in the app
tools = tools_client.list_tools(parent=app_name)
for tool in tools:
    print(f"Tool: {tool.display_name}")

3. Update Operations

Updating resources typically involves retrieving the resource, modifying its properties, and calling an update_* method. SCRAPI often uses field masks to update only specific fields.

# Update agent instructions
agent.instructions = "You help users find products and check their stock status."
updated_agent = agents_client.update_agent(
    agent=agent,
    update_mask="instructions"
)

4. Delete Operations

Deleting is straightforward using delete_* methods and the resource name.

# Delete the tool we created
tools_client.delete_tool(name=new_tool.name)
print("Tool deleted.")

The "Why" from the LLM Perspective

Why do we care about programmatic CRUD from an LLM perspective? Because **reproducibility guarantees behavior**. When we manually click around the UI to create agents or tools, we introduce human variance. An LLM's behavior is highly sensitive to its prompt (instructions) and the tools it has access to. By using scripts to create and update these resources, we ensure that the exact same configuration is deployed across testing, staging, and production environments. This eliminates "it worked in my sandbox" issues and allows us to run large-scale automated evaluations on the exact same agent definition that will serve customers.

Pro-Tip: Idempotency in Scripts

When writing build scripts, make them idempotent. Check if a resource (like an agent or tool with a specific display name) already exists before creating it. If it exists, update it instead of failing or creating duplicates.

Watch-Out: Asynchronous Operations

Some operations in CXAS can be long-running. While SCRAPI handles many of these synchronously by polling for completion, be aware that creating large resources or running complex updates might take a few moments. Don't assume immediate availability in subsequent lines without checking.
