Week 2 Detailed Summary

This week we transitioned from basic agent setup to advanced orchestration and specialized tools. Here is a recap of the core pillars we covered:

1. The Steering Agent Pattern

As agents take on more tasks, a single prompt can become overloaded. The Steering Agent pattern introduces a central "router" or "manager" agent that understands the high-level goal and delegates specific tasks to specialized sub-agents or focused tool sets. This improves accuracy and maintainability by keeping the scope of each agent component narrow.

2. Advanced Callbacks

Callbacks are the hooks that allow you to monitor and intervene in the agent's lifecycle. We explored how to use callbacks not just for logging, but for dynamic behavior modification, such as intercepting tool calls, enriching context on the fly, or enforcing business rules before a response is sent to the user.

3. OpenAPI & Connector Tools

Agents need to interact with the real world. We learned how to bridge the gap between LLMs and external systems using OpenAPI specifications. By defining clear schemas, CXAS can automatically generate connector tools that the agent can use to fetch data or trigger actions in external services securely.

4. Python Code Tools

LLMs are great at language but often struggle with precise, deterministic calculations or complex data manipulation. We integrated Python Code Tools to give the agent a "sandbox" where it can generate and execute Python code. This allows the agent to solve math problems, process data structures, and execute complex algorithms reliably.

5. Tool Chaining & Wrappers

Efficiency is key for user experience. We learned that calling tools sequentially can introduce significant latency. We covered **Tool Chaining** (guiding the agent to sequence calls) and the **Wrapper Pattern** (combining multiple backend operations into a single tool) to minimize execution turns and provide faster responses in complex CUJs.

  1. Understand the user's request to return an item (Steering/CUJ identification).
  2. Check the return policy for that specific item type using a knowledge base.
  3. Calculate the refund amount, deducting a restocking fee if applicable (Deterministic logic).
  4. Call the warehouse API to generate a return shipping label (External API).

  • What tools will you need? (Specify at least one Python Code Tool and one Connector Tool).
  • How will you optimize for latency if the user asks "Can I return this and how much will I get back?"

Pro-Tip: Consolidate with Documentation

Watch-Out: Over-Orchestration

A common pitfall is making the system too complex too early. Only use a Steering Agent if a single agent is failing due to prompt length or confusion. Only build a Wrapper Tool if sequential tool calling is causing noticeable latency. Start simple, measure, and then optimize.
