Solution: Python Script for Simulation Evals

Here is the complete Python script to automate the simulation eval for the "Late Return" scenario. Note how we define the persona, goal, and success criteria, and then programmatically check the result.

import os
from cxas_scrapi import Evaluations

def run_simulation_eval():
    """Triggers a simulation evaluation against the EcoShop agent."""
    project_id = os.environ.get("GCP_PROJECT_ID")
    app_id = os.environ.get("CXAS_APP_ID")
    
    if not project_id or not app_id:
        print("Error: GCP_PROJECT_ID or CXAS_APP_ID environment variables not set.")
        return

    app_name = f"projects/{project_id}/locations/global/apps/{app_id}"
    evals_client = Evaluations()
    
    # 2. Define simulation configuration for the "Late Return" scenario
    simulation_config = {
        "app_name": app_name,
        "user_persona": "A polite but firm customer trying to return shoes bought 45 days ago.",
        "conversation_goal": "Get a refund despite being outside the 30-day window.",
        "max_turns": 6,
        "success_criteria": "The agent must politely refuse the refund based on the policy but offer an alternative like store credit or an exchange."
    }
    
    print("Triggering simulation eval for Late Return scenario...")
    result = evals_client.run_simulation(simulation_config)
    
    # 4. Programmatically check results
    print(f"\nStatus: {result.status}")
    print(f"Score: {result.score}")
    print(f"Reasoning: {result.reasoning}")
    
    if result.status == "PASSED":
        print("Result: SUCCESS - Agent followed policy.")
    else:
        print("Result: FAIL - Agent did not meet success criteria.")
        print("\nConversation Log:")
        for turn in result.conversation_log:
            print(f"{turn.role}: {turn.text}")

    # Advanced: Structured Evaluation Prompt for Simulator Judge
    print("\n--- Advanced: Structured Evaluation Prompt for Simulator Judge ---")
    print("Behind the scenes, the evaluation engine might use a structured prompt like this to judge the conversation:\n")
    
    eval_prompt_template = """
You are an expert quality assurance judge for conversational AI.

Evaluate if the agent correctly followed the return policy in the conversation.


- Base your evaluation strictly on the provided transcript.
- The return policy states items must be returned within 30 days.




Verify adherence to policy:

Agent addresses the return request.
Check if the agent identified that the purchase was outside the 30-day window.


Agent refuses the refund.
Verify the refusal was polite and cited the policy.


Agent offers alternatives.
Check if the agent offered store credit or an exchange instead of a full refund.




Transcript:
{transcript}
"""
    print(eval_prompt_template.format(transcript="[Transcript would appear here]"))

if __name__ == "__main__":
    # Set these in your environment before running:
    # export GCP_PROJECT_ID="your-project-id"
    # export CXAS_APP_ID="your-app-id"
    run_simulation_eval()

Key Steps Explained

  • Dynamic Simulation: Instead of static inputs, we used an LLM-driven persona to converse dynamically with the agent.
  • Structured Evaluation: In the advanced section, we demonstrated how to use <role>, <primary_goal>, <subtask>, and <step> tags to structure the evaluation criteria for the judge. This aligns with best practices for complex prompt engineering.
