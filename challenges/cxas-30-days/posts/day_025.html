Detailed Lesson: Introduction to SCRAPI

The CX Agent Studio Scripting API (**SCRAPI**) is a high-level Python library that extends the official Google Cloud client for CXAS. It is designed to make the lives of bot builders and developers easier by providing a more pythonic and intuitive interface for managing agents, tools, and evaluations.

Why Use SCRAPI?

  • Efficiency: Automate repetitive tasks like creating agents, uploading tools, or updating prompts.
  • Testing: Run complex, multi-turn simulations and turn-based evaluations programmatically.
  • CI/CD: Integrate agent testing and deployment into your standard developer workflows.

Authentication

SCRAPI relies on standard Google Cloud authentication. Before running any scripts, you must ensure your environment is authenticated. Typically, this involves running:

gcloud auth application-default login

This command sets up your Application Default Credentials (ADC), which the SCRAPI library will automatically detect and use.

Your First API Call

Let's look at a basic script to list the applications in your project using the Apps client.

from cxas_scrapi import Apps

# Initialize the Apps client
# It automatically picks up your Google Cloud credentials
apps_client = Apps()

# Define your project ID
PROJECT_ID = "your-project-id"

# List all apps in the project
apps = apps_client.list_apps(project_id=PROJECT_ID)

print("Available Applications:")
for app in apps:
    print(f"- Name: {app.display_name}, ID: {app.name}")

The "Why" from the LLM Perspective

From an LLM perspective, programmatic control via SCRAPI is transformative. Instead of relying on a human to manually test edge cases in a UI, scripts can generate thousands of variations of user inputs and feed them to the agent. This enables automated scaling of evaluations. Furthermore, by using scripts to "shift-left" testing (running unit tests on tools and guardrails before full system tests), we ensure that the LLM is operating on a reliable foundation, reducing non-deterministic errors and hallucinations.

Pro-Tip: Use Service Accounts for Automation

While application-default login is great for local development, always use dedicated Service Accounts with minimal permissions when running SCRAPI scripts in automated pipelines (like GitHub Actions or Cloud Build).

Watch-Out: Hardcoding Project IDs

Avoid hardcoding your Project ID directly in your scripts. Use environment variables to make your scripts portable across different environments (Dev, Staging, Prod).
