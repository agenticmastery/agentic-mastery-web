Solution: Python Script for Automated Testing

Here is the complete Python script to automate the regression test for the "Check Order Status" CUJ. Note how we use unique session IDs for test isolation and include an advanced section showing how to use structured prompts for LLM-as-a-Judge evaluation.

import os
import uuid
from cxas_scrapi import Sessions

def run_regression_tests():
    """Runs automated tests against the EcoShop agent."""
    project_id = os.environ.get("GCP_PROJECT_ID")
    app_id = os.environ.get("CXAS_APP_ID")
    
    if not project_id or not app_id:
        print("Error: GCP_PROJECT_ID or CXAS_APP_ID environment variables not set.")
        return

    app_name = f"projects/{project_id}/locations/global/apps/{app_id}"
    sessions_client = Sessions()
    
    # 2. Define a list of test queries for the "Check Order Status" CUJ
    test_queries = [
        "Where is my stuff?",
        "Order status please",
        "Track order 999"
    ]
    
    print(f"Starting regression tests for App: {app_name}\n")
    
    for i, query in enumerate(test_queries):
        # Best Practice: Use a unique session ID for each test to ensure isolation
        session_id = f"test-order-{uuid.uuid4().hex[:8]}"
        
        print(f"Test {i+1}: User: '{query}'")
        
        response = sessions_client.run(
            app_name=app_name,
            text=query,
            session_id=session_id
        )
        
        print(f"Agent: '{response.text}'")
        
        # 4. Verify response (Basic string matching)
        response_lower = response.text.lower()
        passed = False
        if "order" in response_lower or "status" in response_lower or "number" in response_lower:
            passed = True
            
        if passed:
            print("Result: PASS\n")
        else:
            print("Result: FAIL\n")

    # Advanced: LLM-as-a-Judge Structure
    # Prompt examples in solution use the best practices structure
    print("--- Advanced: LLM-as-a-Judge Prompt Structure ---")
    print("To go beyond simple string matching, use an LLM to judge responses with structured prompts:\n")
    
    eval_prompt_template = """

You are a test evaluator. Your goal is to determine if the agent's response successfully addressed the user's query about order status.



Evaluate the response based on the following criteria:

Did the agent acknowledge the request for order status?


Did the agent either provide the status or ask for a specific order number to look it up?



User Query: {query}
Agent Response: {response}

Output only 'PASS' or 'FAIL' with a short reason.
"""
    print(eval_prompt_template.format(
        query="Where is my stuff?", 
        response="I can help you track your order. Could you please provide your order number?"
    ))

if __name__ == "__main__":
    # Set these in your environment before running:
    # export GCP_PROJECT_ID="your-project-id"
    # export CXAS_APP_ID="your-app-id"
    run_regression_tests()

Key Steps Explained

  • Test Isolation: We used uuid.uuid4() to generate unique session IDs for each turn. This prevents context from leaking between tests.
  • Best Practice Prompts: In the advanced section, we demonstrated how to use <goal>, <subtask>, and <step> tags to structure a prompt for an LLM judge. This ensures the judge understands its role and evaluation criteria clearly.
