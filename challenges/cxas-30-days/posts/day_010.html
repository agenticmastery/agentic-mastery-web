Lesson: Python Code Tools in CXAS

Large Language Models (LLMs) are incredibly powerful at understanding natural language, identifying user intent, and generating human-like responses. However, they are probabilistic models. They are not calculators. When asked to perform complex math or evaluate multi-step logical rules, they can hallucinate or make subtle errors.

To address this, CX Agent Studio (CXAS) allows you to create **Python Code Tools**. You can write custom Python functions directly in the CXAS console. These functions can be called by the agent when it needs to perform precise calculations or data transformations.

How it Works

When you define a Python Code Tool, you provide a function definition and the code itself. You also provide a description for the tool, just like you do for OpenAPI tools. This description is crucial for the LLM to understand when to use the tool.

To make a tool available to an agent, you attach it and can reference it in your prompts or instructions using the syntax: {@TOOL: tool_name}.

The 'Why' from the LLM Perspective

Why not just tell the LLM the rules in the prompt and ask it to calculate? Because LLMs predict the next most likely word, not the mathematically correct answer. For example, calculating a 15% discount on a cart total of $123.45 might lead to an incorrect result if the LLM relies solely on its training data.

By offloading this to a Python Code Tool, the LLM's role changes from *calculator* to *orchestrator*. The LLM extracts the necessary parameters from the conversation (e.g., cart total, discount code), calls the Python tool, and receives the exact, correct answer back from the secure sandbox execution environment. This guarantees accuracy and builds trust with the user.

Secure Sandbox Execution

The Python code you write in the CXAS console does not run on the user's device or directly within the LLM. It executes in a secure, isolated server-side sandbox. This protects the platform and ensures that the code cannot access unauthorized resources or cause harm.

The Day 10 Challenge: EcoShop Shipping Calculator

Scenario: EcoShop needs a reliable way to calculate shipping fees for their agents to quote to customers. The rules are too complex to trust the LLM to calculate them directly in every turn of the Customer User Journey (CUJ).

Task: Write a Python function that calculates the shipping fee based on the following business rules:

  • Base fee: $5.00
  • Weight surcharge: If the package weight is greater than 5 lbs, add $2.00 for every pound (or fraction thereof) above 5 lbs.
  • Destination surcharge: International shipping adds a flat $15.00 to the total.
  • Loyalty discount: Gold members get 20% off the total shipping fee. Silver members get 10% off. Regular members get no discount.

Your function should accept parameters for weight, destination_type (e.g., 'domestic' or 'international'), and loyalty_tier (e.g., 'regular', 'silver', 'gold').

Write the Python code snippet that you would enter into the CXAS console. Do not provide the answer here; focus on implementing the logic correctly in code.

Difficulty: Medium. Ensure you handle the weight surcharge calculation correctly for weights above 5 lbs.

Pro-Tip: Keep Functions Pure

Ensure your Python functions are deterministic (pure). Given the same inputs, they should always return the same output. Avoid relying on external state or random number generators within these tool functions to ensure consistent agent behavior.

Watch-Out: Sandbox Limitations

The secure sandbox environment has limitations. You cannot import arbitrary third-party libraries that are not pre-approved by the platform. Stick to standard Python libraries for your logic to ensure it runs without errors.
