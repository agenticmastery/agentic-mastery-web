Detailed Lesson: Web Deployment with Agent Kite

Deploying your agent to the web is the process of making it available to users through a chat interface on your website. Agent Kite provides a powerful and customizable web widget (also known as the test kit) that makes this process straightforward.

The deployment typically involves two main steps:

  1. Generating the Snippet: The platform generates a small snippet of HTML and JavaScript code that references the Agent Kite library and includes configuration specific to your agent.
  2. Embedding the Snippet: You copy this snippet and paste it into the HTML of the web pages where you want the chat widget to appear.

The configuration snippet usually includes parameters like the agentId, conversationId (optional for starting new sessions), and styling options to match your brand.

The 'Why' from the LLM Perspective

From the perspective of the LLM and the backend platform, the web widget is more than just a UI component; it is a state manager and a bridge. LLMs are inherently stateless—they don't remember previous turns unless that history is provided with each new prompt. The Agent Kite widget handles this by maintaining a persistent session.

When a user starts a chat, the widget initiates a session and receives a session ID. For every subsequent message the user sends, the widget attaches this session ID. This allows the platform to retrieve the conversation history (the full transcript of the Customer User Journey or CUJ) and feed it back to the LLM, ensuring contextually relevant responses. Without the widget managing this session state, the LLM would treat every message as a completely new interaction, losing all context.

Furthermore, the widget handles the secure connection to the platform, ensuring that user inputs are securely transmitted and responses are received and rendered in real-time, handling aspects like typing indicators and rich responses seamlessly.

Pro-Tip: Asynchronous Loading

Always load the chat widget script asynchronously. This ensures that the loading of the chat interface does not delay the rendering of your main website content, providing a better user experience.

Watch-Out: Session Persistence

Ensure your widget is configured to handle session persistence correctly. If a user refreshes the page or navigates to another page on your site, the chat session should ideally continue rather than starting over. Check the Agent Kite documentation for session storage options (e.g., local storage vs. session storage).
