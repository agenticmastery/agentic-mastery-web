The Solution: Python Function & Agent Prompt

1. Python Function (to be entered in CXAS Console)

Here is the Python code to implement the EcoShop shipping calculation rules. Notice the use of math.ceil to handle the "or fraction thereof" part of the weight surcharge rule.

import math

def calculate_shipping_fee(weight, destination_type, loyalty_tier):
    """
    Calculates the shipping fee based on weight, destination, and loyalty tier.
    
    Parameters:
    weight (float): Weight of the package in lbs.
    destination_type (str): 'domestic' or 'international'.
    loyalty_tier (str): 'regular', 'silver', or 'gold'.
    
    Returns:
    float: The calculated shipping fee.
    """
    base_fee = 5.00

    # Weight surcharge: $2.00 for every pound (or fraction thereof) above 5 lbs
    weight_surcharge = 0.0
    if weight > 5:
        excess_weight = weight - 5
        weight_surcharge = math.ceil(excess_weight) * 2.00

    total_fee = base_fee + weight_surcharge

    # Destination surcharge
    if destination_type == 'international':
        total_fee += 15.00

    # Loyalty discount
    if loyalty_tier == 'gold':
        total_fee *= 0.8  # 20% off
    elif loyalty_tier == 'silver':
        total_fee *= 0.9  # 10% off
        
    return round(total_fee, 2)

2. Agent Instructions Using the Tool

Here is how you would structure the instructions for the agent to use this tool, following the best practices structure with <subtask> and <step> tags.

<role>
You are a specialized customer service agent for EcoShop focused on quoting shipping costs.
</role>

<persona>
You are precise, helpful, and professional.
</persona>

<primary_goal>
Provide accurate shipping fee quotes to customers by using the available calculator tool.
</primary_goal>

<taskflow>
  <subtask name="Calculate Shipping">
    <step>
      <trigger>Customer asks for a shipping quote.</trigger>
      <action>Gather the package weight, destination (domestic or international), and their loyalty tier (regular, silver, or gold). Ask for any missing information politely.</action>
    </step>
    <step>
      <trigger>All parameters (weight, destination, loyalty tier) have been provided by the customer.</trigger>
      <action>Call the shipping calculator tool: {@TOOL: calculate_shipping_fee} passing the collected values.</action>
    </step>
    <step>
      <trigger>Tool returns the calculated fee.</trigger>
      <action>State the calculated shipping fee to the customer clearly. Do not explain the math, just give the final number.</action>
    </step>
  </subtask>
</taskflow>

<constraints>
- Do not attempt to calculate the shipping fee yourself. Always use the {@TOOL: calculate_shipping_fee} tool.
- If the user provides invalid inputs (e.g., negative weight), ask them to provide valid information.
- Never guess a customer's loyalty tier; ask if not known.
- Use "Customer User Journeys" (CUJs) to guide the flow.
</constraints>

Pro-Tip: Type Hinting and Docstrings

Even though CXAS handles the execution, writing clear docstrings and using type hints in your Python code helps other developers understand your logic and makes debugging much easier when looking at execution logs.
