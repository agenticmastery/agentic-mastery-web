Solution: EcoShop Voice Integration

Here is the proposed solution for mapping the EcoShop agent to a Twilio number and handling order status inquiries.

1. Telephony Configuration (Twilio)

To connect the agent to a Twilio number, you would typically follow these steps in the CXAS console and Twilio console:

  1. In CXAS: Navigate to the Deployment section and select "Telephony". Choose "Twilio" as the provider. This will generate a unique endpoint URL (Webhook) for your agent.
  2. In Twilio: Go to your Phone Numbers console and select the number you want to use.
  3. Configure Webhook: In the "A CALL COMES IN" section of the number configuration, select "Webhook" and paste the endpoint URL provided by CXAS. Ensure the HTTP method is set to POST.
  4. Enable Streaming: If supported, enable bi-directional streaming to take advantage of CXAS's low-latency architecture.

2. Voice-Optimized Prompt Design

Here is an example of how you might structure the prompt for the agent to handle order status inquiries over the phone, using best practices for voice and the required structure.

<subtask>Handle order status inquiry via voice</subtask>
<step>Ask the user for their order number. Keep the request brief, e.g., "Sure, I can help with that. What is your order number?"</step>
<step>Once the user provides the number, call the tool {@TOOL: check_order_status} with the order_id.</step>
<step>If the tool returns the status, read it back to the user concisely. For example, "Your order is currently being processed and should ship tomorrow."</step>
<step>Ask if there is anything else they need help with, keeping it brief: "Is there anything else I can help you with?"</step>

Key Considerations for Voice:

  • Conciseness: Notice that the prompts are short. We don't say "Thank you for calling EcoShop, the best place for eco-friendly products. I would be delighted to help you check your order status today if you could kindly provide me with your order number." We get straight to the point.
  • Turn-Taking: The steps are designed to prompt the user for specific information and then wait for a response, minimizing overlapping speech.
