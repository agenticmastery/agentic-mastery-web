Solution: Python Script for One-Click Deploy

Here is the complete Python script to automate the creation of the agent, registering the {@TOOL: GetOrderDetails} tool, binding it, and running a verification test.

import os
from cxas_scrapi import Apps, Tools, Evaluations

def run_one_click_deploy():
    """Automates the creation, tool binding, and testing of an agent."""
    project_id = os.environ.get("GCP_PROJECT_ID")
    location = "global"
    
    if not project_id:
        print("Error: GCP_PROJECT_ID environment variable not set.")
        return

    apps_client = Apps()
    tools_client = Tools()
    evals_client = Evaluations()

    # 1. Create Agent with best-practice structured instructions
    print("Creating agent...")
    instructions = """
You are a professional customer service agent for EcoShop.

Help customers track their orders and answer questions politely.



Handle order tracking:

Customer asks "Where is my order?" or provides an order ID.
Call the {@TOOL: GetOrderDetails} tool to retrieve status.



"""
    
    app_config = {
        "display_name": "EcoShop-FullAuto-SR",
        "instructions": instructions,
        "model": "gemini-1.5-pro"
    }
    
    try:
        app = apps_client.create_app(project_id, location, app_config)
        app_name = app.name
        print(f"Agent created: {app_name}")

        # 2. Create Tool
        print("Creating GetOrderDetails tool...")
        tool_config = {
            "display_name": "GetOrderDetails",
            "description": "Retrieves shipping status and items for a given order ID.",
            "function_spec": {
                 "name": "get_order_details",
                 "parameters": {
                     "type": "OBJECT",
                     "properties": {
                         "order_id": {"type": "STRING", "description": "The 6-digit order number."}
                     },
                     "required": ["order_id"]
                 }
            }
        }
        tool = tools_client.create_tool(project_id, location, tool_config)
        print(f"Tool created: {tool.name}")

        # 3. Bind Tool to App
        print("Binding tool to agent...")
        apps_client.bind_tool(app_name, tool.name)
        print("Tool bound successfully.")

        # 4. Run Baseline Test
        print("Running verification simulation...")
        simulation_config = {
            "app_name": app_name,
            "user_persona": "A customer checking on their order.",
            "conversation_goal": "Ask 'Where is my order #12345?'",
            "max_turns": 4,
            "success_criteria": "The agent must attempt to call the {@TOOL: GetOrderDetails} tool with the order_id '12345'."
        }
        result = evals_client.run_simulation(simulation_config)
        
        print(f"\nTest Status: {result.status}")
        print(f"Score: {result.score}")
        print(f"Reasoning: {result.reasoning}")
        
        if result.status == "PASSED":
            print("\nBuild and Test SUCCESSFUL!")
        else:
            print("\nBuild succeeded but Test FAILED.")
            
    except Exception as e:
        print(f"\nAn error occurred during deployment: {e}")
        print("Consider implementing cleanup logic here to remove partially created resources.")

if __name__ == "__main__":
    run_one_click_deploy()

Key Steps Explained

  • Structured Instructions: We used <role>, <primary_goal>, <subtask>, and <step> tags in the agent's instructions. This helps the LLM understand its task flow clearly.
  • Sequential Orchestration: The script ensures that the tool is created and bound *before* the test is run, preventing race conditions.
  • Error Handling: The try...except block catches failures, and a production script should include specific cleanup calls in a finally block.
