Recommended Solution: Guardrail Configuration

Here is a recommended approach to configuring the guardrail instructions for the EcoShop agent, using a structured prompt format to ensure clear execution by the LLM.

Structured Guardrail Instructions

We use a structured format with `` and `` tags to make the instructions unambiguous for the model.


<subtask name="Guardrail: Competitor Refusal">
    <step number="1">Identify if the user is asking to compare EcoShop products with competitors (e.g., "MegaMart") or asking for recommendations of other brands.</step>
    <step number="2">If a competitor discussion is detected, refuse to answer politely.</step>
    <step number="3">Respond with: "I can only provide information about EcoShop products and services. I am not able to compare them with other brands. How else can I help you today?"</step>
</subtask>

<subtask name="Guardrail: Out-of-Scope Chitchat Refusal">
    <step number="1">Identify if the user's query is unrelated to EcoShop products, services, returns, or order management (e.g., questions about weather, sports, general knowledge, or personal advice).</step>
    <step number="2">If the query is out-of-scope, refuse to answer politely.</step>
    <step number="3">Respond with: "I am designed to assist with EcoShop inquiries. I cannot help with unrelated topics. Is there an EcoShop product or order I can help you with today?"</step>
</subtask>

Key Considerations in this Solution

  • **Explicit Triggers:** The steps clearly define what to look for (competitor names, unrelated topics).
  • **Standardized Responses:** Providing a specific response template ensures consistency and prevents the model from improvising potentially unsafe or off-brand answers.
  • **Fallback to Supported CUJs:** The refusal responses end by inviting the user back to valid Customer User Journeys (CUJs) supported by EcoShop.

Calibrating with Guardrail Evals

After implementing these instructions, you should use **Guardrail Evals** (via SCRAPI) to test them. Create a dataset with queries like:

  • *Safe:* "How do I return my EcoShop shoes?"
  • *Malicious/Out-of-Scope:* "Is MegaMart cheaper than EcoShop?"
  • *Chitchat:* "What's the weather like in New York?"

Run the eval to ensure the agent refuses the bad ones and answers the good ones, adjusting the instructions or thresholds as needed.
