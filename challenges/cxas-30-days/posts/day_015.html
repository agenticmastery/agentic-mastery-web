Detailed Lesson: Guardrails in CX Agent Studio

In CX Agent Studio, **Guardrails** are policies you put in place to ensure the agent behaves safely and stays within its intended scope. They are essential for preventing the agent from hallucinating on sensitive topics, discussing competitors, or engaging in inappropriate conversations.

Configuring Guardrails for Out-of-Scope Handling

One of the most common uses of guardrails is to define what is **out-of-scope**. For example, an agent designed to help with retail orders shouldn't be answering questions about the weather or giving medical advice.

To configure guardrails effectively, you need to:

  • **Define Clear Boundaries:** Specify exactly what topics are allowed and what are forbidden. Instead of relying on broad "Intents", focus on the specific **Customer User Journeys (CUJs)** the agent is authorized to handle.
  • **Provide Refusal Instructions:** Tell the agent exactly how to respond when a user crosses a boundary. The response should be polite but firm, redirecting the user back to supported topics.
  • **Calibrate Refusal Thresholds:** Guardrails often use a separate classification step or specific instructions to decide when to refuse. You need to balance safety (refusing bad prompts) with helpfulness (not refusing valid prompts that might sound slightly off).

Guardrail Evals

As mentioned in the SCRAPI guide, CXAS supports **Guardrail Evals**. These are designed for security tuning. You can run sample datasets containing both safe and malicious/out-of-scope prompts against your offline Guardrail configurations. This allows you to measure precision and recall for refusals and calibrate the thresholds effectively before going live.

The 'Why' from the LLM Perspective

Why do we need a separate guardrail system instead of just putting "don't talk about competitors" in the main prompt? Large Language Models are trained to be helpful and follow instructions, which makes them susceptible to "prompt injection" or being led astray by clever users. By implementing guardrails, often as a separate filter on model attention or output, we add a layer of defense. This separate mechanism evaluates the input (or output) against the safety policy before the main generation or delivery, ensuring that even if the model *wants* to answer an out-of-scope question, the system prevents it from doing so.

The Day 15 Challenge: Guardrail Configuration for EcoShop

Scenario: EcoShop wants to ensure its customer service agent stays focused on helping customers with EcoShop products and services. Management is concerned that users might ask the agent to compare EcoShop products with competitors (like "MegaMart") or just use the agent for casual chitchat.

Task: Define the guardrail rules for the EcoShop agent to prevent out-of-scope interactions. Your configuration should address:

  • **Competitor Discussion:** How should the agent handle questions about "MegaMart" or general comparisons?
  • **Chitchat/General Knowledge:** How should the agent handle questions about the weather, sports, or general advice?

In your response, outline the **criteria** for what constitutes out-of-scope in these areas and the **refusal strategy** (what the agent should say). Do not write the exact prompt instructions, just the high-level rules.

Pro-Tip: Keep Guardrails Explicit and Focused

Avoid vague rules like "be helpful and polite". Instead, use explicit negative constraints like "DO NOT discuss competitor pricing" or "DO NOT answer questions unrelated to order management or product inquiries." Clear, specific rules are much easier for the model (and the guardrail evaluator) to follow.

Watch-Out: Over-Restricted Agents

Be careful not to make your guardrails too strict. If a customer mentions a competitor casually ("I used to shop at MegaMart but I like your eco-friendly focus"), the agent shouldn't shut them down aggressively. It should acknowledge the context and stay focused on the EcoShop value proposition. Over-restricting can lead to a frustrating user experience where the agent refuses to answer legitimate questions that happen to touch on a restricted keyword.
