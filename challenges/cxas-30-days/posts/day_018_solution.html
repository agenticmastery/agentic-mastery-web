Answers & Reference Strategy

Part 1: Reflection & Strategy (Loyalty Upgrade Feature)

Here are the recommended approaches for the strategy exercise:

1. Edge Cases for Golden Dataset:

  • Negative Case: A customer trying to upgrade with insufficient points. The agent should politely explain why and what is needed.
  • Boundary Case: A customer with exactly the minimum points required. This tests precise logic.
  • Interrupted Journey: A customer who starts the upgrade process but suddenly asks an unrelated question (e.g., "Wait, what is your return policy?"). The agent should answer and then return to the upgrade flow.
  • Security Case: A user attempting to upgrade an account they are not authorized for (handled via authentication checks, but the agent prompt should handle the denial gracefully).

2. Critical Component for Isolation Testing:

The tool that calculates the points and applies the upgrade (e.g., {@TOOL: process_loyalty_upgrade}) is the most critical. It directly affects the database state and business value. You must ensure that given a specific user state, it computes the correct output every time, without relying on the LLM's reasoning.

3. Essential Guardrail:

A guardrail that explicitly checks for and blocks any generated text where the agent promises a free upgrade or discounts without the appropriate tool execution confirmation. You could also use a hard constraint in the prompt, but a guardrail provides the secondary safety net.

4. Latency Monitoring:

You should measure the full turn latency for the upgrade operation. A typical p99 target for a complex operation involving multiple tool calls might be around **3.0 to 4.0 seconds**, but the target should always be as low as possible while maintaining accuracy. You would look at trace reports to see if database calls are the bottleneck.

Part 2: Knowledge Check (MCQs) Answers

  1. Which evaluation method is best for testing how an agent handles a complex, multi-step Customer User Journey (CUJ)?
    Answer: C) Simulation Evaluation using Golden Datasets.
    Explanation: Simulation evals test the entire flow across multiple turns, which is necessary to see if context is maintained and the goal is reached.
  2. Why do we prioritize fixing high p99 latency even if the p50 latency is excellent?
    Answer: B) Because high p99 means the worst-case experiences are unacceptably slow, risking high frustration for some users.
    Explanation: The p99 metric represents the 1% of users having the worst experience. Ignoring it means ignoring a segment of users who may churn due to poor performance.
  3. What is the primary purpose of PII Redaction in an agentic system?
    Answer: B) To prevent sensitive user data from being exposed in logs or processed insecurely.
    Explanation: PII redaction is a security measure to protect user privacy and comply with regulations like GDPR or HIPAA.
  4. Isolation testing of a tool involves:
    Answer: B) Testing the tool function directly with specific inputs and mocked external dependencies, bypassing the LLM.
    Explanation: Isolation testing treats the tool like a standard unit test, removing the non-deterministic LLM from the equation.
