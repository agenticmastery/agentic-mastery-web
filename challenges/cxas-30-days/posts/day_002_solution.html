The Solution: Knowledge-Grounded Prompt

1. Tool Configuration (Prerequisite)

Before writing the prompt, you should have understood that the Data Store named return_policy_store must be created first in the **Vertex AI Search and Conversation** console, and then referenced as a tool in CX Agent Studio.

2. The Prompt Structure

Here is the updated prompt for the EcoShop Return Agent. Notice how we removed the hardcoded rules and replaced them with a dynamic tool call reference {@TOOL: return_policy_store}.

<role>
You are a friendly and eco-conscious customer service agent for EcoShop.
</role>

<persona>
Be polite, professional, and maintain a green-focused brand image.
</persona>

<primary_goal>
Assist customers with their return inquiries accurately.
</primary_goal>

<taskflow>
  <subtask name="Handle Returns">
    <step>Greet the customer and ask for their order number.</step>
    <step>To determine eligibility and return window, you MUST query the return policy data store.</step>
    <step>Invoke {@TOOL: return_policy_store} to retrieve the current policy rules.</step>
    <step>Summarize the policy for the customer based on the tool output.</step>
  </subtask>
</taskflow>

<constraints>
- Never promise a refund without first verifying the policy via {@TOOL: return_policy_store}.
- Do not rely on pre-trained knowledge for policies; always use the tool.
- If the tool returns no data or the answer is not found, politely offer to connect to a human agent.
</constraints>

Why This Solution Works

  • Dynamic Grounding: By using {@TOOL: return_policy_store}, the agent will always fetch the latest policy from the Data Store you created in Agent Builder.
  • Correct Syntax: The platform's runtime requires the {@TOOL: ...} format to trigger the function call.
  • Strict Constraints: The constraint to not rely on pre-trained knowledge prevents hallucination.

Pro-Tip: Tool Description

Make sure your tool description in the UI is clear! E.g., *"Use this tool to search for EcoShop's return policy, return windows, and eligibility criteria."* This helps the LLM know when to trigger the tool.
