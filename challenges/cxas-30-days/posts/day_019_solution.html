Solution for embedding the Agent Kite web widget.

Challenge Recap

You were asked to describe the steps to embed the Agent Kite web widget for EcoShop, ensuring asynchronous loading, passing the agentId "ecoshop-support-v1", and setting a custom title "EcoShop Support".

The Solution

Here is a typical implementation for embedding the Agent Kite web widget. This snippet should ideally be placed just before the closing </body> tag of your HTML file to ensure it doesn't block the rendering of critical page content.

<!-- Agent Kite Web Widget -->
<script>
  window.agentKiteSettings = {
    agentId: 'ecoshop-support-v1',
    title: 'EcoShop Support',
    autoOpen: false // Best practice: let the user initiate
  };
</script>
<script async src="https://cdn.agentkite.com/widget.js"></script>

Explanation of the Code

  • window.agentKiteSettings: This object holds the configuration for the widget. The library reads these settings when it loads.
  • agentId: Identifies your specific agent instance in the backend.
  • title: Customizes the header of the chat window.
  • async: The async attribute on the <script> tag ensures the script is fetched asynchronously, allowing the browser to continue parsing the HTML without waiting for the script to download.

Best Practices Applied

  • Placement: Placing the script at the bottom of the body is a standard performance optimization.
  • Asynchronous Loading: Prevents the widget from becoming a render-blocking resource.
  • User-Initiated: Setting autoOpen: false is often preferred so as not to annoy users, though some scenarios might require auto-opening.

Pro-Tip: Custom Styling

Most enterprise widgets allow you to override CSS variables to match your brand colors. Check the documentation for properties like --agent-kite-primary-color to customize the look and feel without editing the library code.
